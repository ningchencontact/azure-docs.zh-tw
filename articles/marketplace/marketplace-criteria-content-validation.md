# <a name="azure-marketplace-listing-guidelines"></a>Azure Marketplace 列名指導方針

本文件包含適用於檢閱者針對列名至 Azure Marketplace 之供應項目和服務的需求指導方針與檢查清單。

所有供應項目都必須符合以下所示的[所有列名需求](https://docs.microsoft.com/azure/marketplace/marketplace-criteria-content-validation#all-listing-requirements)。 針對特定列名類型，會提供其他需求和檢查清單，包括：附加[試用版](https://docs.microsoft.com/azure/marketplace/marketplace-criteria-content-validation#trial-offer-requirements)的供應項目、[SaaS 應用程式](https://docs.microsoft.com/azure/marketplace/marketplace-criteria-content-validation#saas-apps-requirements)、[容器](https://docs.microsoft.com/azure/marketplace/marketplace-criteria-content-validation#container-offer-requirements)和[諮詢供應項目](https://docs.microsoft.com/azure/marketplace/marketplace-criteria-content-validation#consulting-offer-requirements)。


## <a name="all-listing-requirements"></a>所有列名需求

|  | 列名項目 | 基本需求 | 最佳需求 |
|:--- |:--- |:--- |:--- |
| 1 | 潛在客戶目的地 | 已設定潛在客戶目的地| 具有潛在客戶目的地 CRM 資訊的 OCP 目錄也列在合作夥伴解決方案索引標籤中 | 
| 2 | 供應項目標題 | 描述解決方案供應項目。 符合合作夥伴網站中的解決方案線上推廣內容 | 包含搜尋關鍵字 |
| 3 | 標誌 | 正確地顯示標誌 | 正確地顯示標誌，並包含[主圖影像](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-solution-template-offer-publish)，其為 Azure 入口網站中的大型格式影像 |
| 4 |  供應項目描述 | <br>• 2 到 3 個段落 <br>• 一目了然的解決方案供應項目 <br>• 供應項目描述沒有任何拼字和文法錯誤 <br>• 供應項目描述詳盡且能緊扣：目標受眾、使用者類型、為何有價值 (價值主張) <br>• 供應項目描述採用段落敘述形式搭配容易理解的簡短句子 | <br>• 目標產業概述 (如果有相關) <br>• 良好的樣式格式，每個段落標題都有一個句子或片語概述後續的內容，且包含項目符號 (如果適當時) 以強調重要的優點。 此處的目標是讓閱讀者能以易於檢視的格式一眼就了解供應項目，而不需要閱讀段落內容。 <br>• 每個段落之間都有間距，且讀起來像汽車手冊。 意思並不是要描述技術功能或程式碼術語，而是以簡單的字詞和完整的描述來說明供應項目。 | 
| 5 | 類別和產業 | <br>• 類別與解決方案供應項目功能相符 <br>• 不會延伸至與解決方案不符的類別 <br>• 最適合的產業，或所有選取的產業 (如果沒有最適合的產業) | <br>• 最多 3 個類別 <br>• 最多 3 個產業 |
| 6 | 映像 | <br>• 影像並非必要，但如果有提供，則必須顯示且為高解析度影像。 <br>• 影像需求列在 CPP 中以及此處 <br>• 包含在螢幕擷取畫面中的文字是清楚易於閱讀的影像 | 解決方案供應項目是一目了然的 |
| 7 | 影片 | <br>• 影片並非必要，但有提供，則必須能播放而不會發生任何錯誤。 <br>• 如果有提供，不得提及競爭對手公司，「除非」是示範移轉解決方案 |<br>• 理想長度為 3 分鐘以上 <br>• 可透過影片內容輕鬆了解解決方案供應項目 <br>• 解決方案功能的示範 |
| 8 | 列名狀態 (行動呼籲) | <br>必須以下列類型之一標示： <ul>o 與我聯絡</ul><ul>o 試用版/立即取得試用版/開始試用/試用產品</ul><ul>o 立即購買/立即取得</ul> | 客戶可以輕易地了解有哪些可用的後續步驟： <ul>1.嘗試試用版</ul> <ul>2.立即購買</ul><ul>3.透過電子郵件或電話號碼連絡以安排概念驗證 (PoC)、評量或簡報。</ul> | 
| 9 | 解決方案價格 | 必須有解決方案價格索引標籤/詳細資料，且以合作夥伴解決方案供應項目的當地貨幣為單位 | 使用定價層提供多種計費選項供客戶選擇 |    
| 10 | 詳細資訊 | 在底部提供有關解決方案詳細資訊的連結 (在描述的底下，而非左側的 Marketplace 連結)，且連結為公開可用並正確顯示 |   連結至特定項目 (例如合作廠商網站上的規格頁面)，而不只是合作夥伴首頁網站 | 
| 11 | 解決方案支援與說明 | 提供至少下列一個的連結： <br>o   電話號碼 <br>o 電子郵件支援 <br>o 聊天代理程式 <br>o   社群論壇 |<br>• 列出所有支援方法：電話與電子郵件支援。 <br>• 在試用版或試用產品期間提供免費的付費支援 | 
| 12 | 法律 | 透過公開 URL 提供原則或條款 |  |

## <a name="saas-apps-requirements"></a>SaaS 應用程式需求

|  | 列名項目 | 基本需求 | 最佳需求 | 
|:--- |:--- |:--- |:--- |
| 1 | 供應項目標題 |<br>• 只能包含小寫英數字元、連字號或底線。 一旦發行後就無法修改。 <br>• 描述解決方案供應項目。<br>• 符合合作夥伴網站中的解決方案線上推廣內容。 | 包含搜尋關鍵字 |
| 2 | 技術資訊：組態 |<br>• 對於 SaaS 應用程式，選擇您要只列出您的應用程式，或是讓客戶可透過 Azure 購買您的應用程式。<br>• 選取要在供應項目取得按鈕上顯示的適當文字：免費、免費試用或與我連絡<br>• 如果您的應用程式利用下列其中一個技術，請只選取 (快顯方塊中) 一個適用的產品：Cortana Intelligence、Power BI 解決方案範本、Power Apps |  |
| 3 | 試用產品 | 選取：是或否 | 客戶可以輕易地了解有哪些可用的後續步驟： <br>1.嘗試試用版 <br>2.立即購買 <br>3.透過電子郵件或電話號碼連絡以安排概念驗證 (PoC)、評量或簡報。 |
| 4 | 店面詳細資料：供應項目摘要 | 這會顯示在您應用程式的搜尋頁面，上限 100 個字元 | |
| 5 | 店面詳細資料：產業 | 產業 (最多 2 個)：選取您應用程式最吻合且最適合的產業。 | |
| 6 | 供應項目描述 | <br>• 允許簡單的 HTML，包括 p、em、ul、li、ol 和標題標記。 上限 3000 個字元。 <br>• 2 到 3 個段落 <br>• 解決方案供應項目是一目了然的 <br>• 描述詳盡且能緊扣：目標受眾*、使用者類型、為何有價值 (價值主張) <br>• 供應項目描述採用段落敘述形式搭配容易理解的簡短句子。 | <br>• 目標產業概述 (如果有相關) <br>• 良好的樣式格式，每個段落標題都有一個句子或片語概述後續的內容，且包含項目符號 (如果適當時) 以強調重要的優點。 此處的目標是讓閱讀者能以易於檢視的格式一眼就了解供應項目，而不需要閱讀段落內容。 <br>• 每個段落之間都有間距，且讀起來像汽車手冊。 意思並不是要描述技術功能或程式碼術語，而是以簡單的字詞和完整的描述來說明供應項目。 | 
| 7 | 行銷成品 | 正確地顯示標誌 | <br>• 標誌包含「主圖影像」，其為 Azure 入口網站中的大型格式影像 <br>• 標誌：必須有小型 (48x48) 和大型 (216x216) <br>• 主圖影像需求： https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-solution-template-offer-publish <br>• 螢幕擷取畫面 (最多 5 個)：CPP 要求 .png 影像，解析度 1280 x 720。 |
| 8 | 類別和產業 | <br>• 類別與解決方案供應項目功能相符 <br>• 應從快顯方塊中選擇至少 1 個項目 <br>• 不會延伸至與解決方案不符的類別 <br>• 最適合的產業，或所有選取的產業 (如果沒有最適合的產業) <br>• 不會延伸至與解決方案不符的類別 <br>• 最適合的產業，或所有選取的產業 (如果沒有最適合的產業) | 最大 3 個選取的類別 (如果適用)。 |
| 9 | 潛在客戶管理 | 選取將儲存您潛在客戶的系統。 在[這裡](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-get-customer-leads)了解如何連線 CRM 系統 |  |
| 10 | 連絡人：解決方案支援與說明 | <br>• 工程連絡人姓名：輸入您應用程式工程連絡人的姓名。 這名連絡人將收到來自 Microsoft 的技術通訊。<br>• 工程連絡人電子郵件：輸入您應用程式工程連絡人的電子郵件地址。<br>• 工程連絡人電話：輸入工程連絡人的電話號碼。 支援 ISO 電話號碼標記法，如需詳細資料，請參閱 https://en.wikipedia.org/wiki/E.123。<br>• 支援連絡人姓名：輸入您應用程式支援連絡人的姓名。 這名連絡人將收到來自 Microsoft 的支援相關通訊。<br>• 支援連絡人電子郵件：輸入您應用程式支援連絡人的電子郵件地址。 <br>• 支援連絡人電話：輸入支援連絡人的電話號碼。 支援 ISO 電話號碼標記法，如需詳細資料，請參閱 https://en.wikipedia.org/wiki/E.123。 <br>• 支援 URL：輸入支援頁面的 URL。 | <br>• 列出所有支援方法：電話與電子郵件支援。 <br>• 在試用版或試用產品期間提供免費的付費支援 |
| 11 | 法律 |<br>• 隱私權原則 URL：輸入應用程式的隱私權原則 URL ( CPP 中隱私權原則 URL 欄位的內容)。 <br>• 使用規定：輸入您應用程式的使用條款。 客戶必須先接受這些條款，才能試用您的應用程式。 |  透過公開 URL 網站提供原則或條款 |

## <a name="consulting-offer-requirements"></a>諮詢供應項目需求
|  | 列名項目 | 基本需求 | 最佳需求 |
|:--- |:--- |:--- |:--- |
| 1 | 供應項目標題  |  <br>• 必須以下列格式清楚地列出服務類型和持續時間：名稱：持續期間類型。 (亦即「供應項目業務開發：1 週概念證明」)  | <br>• 不重複的發行者名稱 <br>• 描述中已提供鏡像標題樣式，不需要長標題。 | 
| 2 | 供應項目描述  |<br>• 確定適當地使用 [Microsoft 產品名稱](https://www.microsoft.com/en-us/legal/intellectualproperty/trademarks/en-us.aspx) <br>• 供應項目註明價格：估價必須在供應項目的頂端或底部註記以說明變化性 (到用戶端出差、要遷移的伺服器數量等等) <br>• 每個供應項目都有描述需求，如下所示： <ul><br>o 簡報中針對簡報涵蓋的主題資訊至少需要 4 到 5 個項目符號來說明 </ul> <ul><br>o 所有工作坊都需要日程表 </ul><ul><br>o 所有日程表都必須細分以日或週為單位，視工作坊的持續時間而定 </ul><ul><br>o 評量、POC、非訓練型工作坊，實作的供應項目則需要交付項目 </ul><ul><br>o 工作坊不需要交付項目，但針對將涵蓋的主題需要更詳細的日程表。 | <br>• 任何供應項目都有日程表和交付項目 <br>• 供應項目頂端區段包含一個段落，含有公司提供之服務的相關內容 <br>• 供應項目包含本身服務之價值的段落，作為頂端區段 |
| 3 | Markdown 格式  | 所有供應項目都必須使用 Markdown 格式，如此在轉換為 HTML 時才會正確轉譯供應項目  |  |
| 4 | 類別和產業 | 不相關的類別 |  | 
| 5 | 列名狀態 (CTA) | 自動列為與我連絡 |  | 
| 6 | 解決方案支援與說明  | 支援與說明為非必要  | | 
| 7 | 提供隱私權原則及使用條款  |  原則或條款為非必要  | | 
| 8 | 服務類型 | 確定服務類型與標題相符 | |
| 9 | 專長認證 |<br>• 必須至少為下列其中一個：<br>• 應用程式開發 <br>• 應用程式整合 <br>• 應用程式生命週期管理 <br>• 雲端平台 <br>• 資料分析 <br>• 資料中心 <br>• 資料平台 <br>• DevOps |
| 10 | 產品 | 必須是 Azure | |
| 11 | 國家/區域 | 確定國家和地區與選擇的貨幣相符 | |
| 12 | 詳細資訊  | <br>• 在底部提供有關解決方案詳細資訊的連結 (在描述的底下，而非左側的 Marketplace 連結)，且連結為公開可用並正確顯示。 <br>• 連結必須有「易記名稱」，且透過任何方式下載都不會顯示為檔案名稱 | |
