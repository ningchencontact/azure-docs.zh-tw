## <a name="access-your-account-settings"></a>存取您的帳戶設定

如果您尚未這樣做，您 （或貴組織的系統管理員） 應該存取[帳戶設定](https://partner.microsoft.com/dashboard/account/management)合作夥伴中心帳戶才能：
- 請檢查您的公司帳戶的驗證狀態
- 確認您的賣方識別碼、 MPN 識別碼、 發行者識別碼和連絡資訊，包括公司核准者與賣方連絡人
- 設定貴公司的財務詳細資料，包括稅額豁免，如果適用的話
- 建立會使用您的商務帳戶在合作夥伴中心內的任何人的使用者帳戶

### <a name="open-developer-settings"></a>開啟開發人員設定

帳戶設定是位在右上角您[商業 Marketplace 儀表板](https://partner.microsoft.com/dashboard/commercial-marketplace)在合作夥伴中心。 選取齒輪圖示 （附近的儀表板右上角），然後選取 **開發人員設定**。 

![在合作夥伴中心內的帳戶設定 功能表](./media/dashboard-developer-settings.png)

內部**帳戶設定**，您將能夠檢視您：
- **帳戶詳細資料**:帳戶類型和帳戶狀態
- **發行者識別碼**:賣方識別碼、 使用者識別碼、 發行者識別碼、 Azure AD 租用戶、 等等。
- **連絡資訊**:發行者顯示名稱、 賣方連絡人姓名、 電子郵件、 電話和地址
- **財務細節**:支付帳戶、 稅金設定檔，並支付保留狀態
- **裝置**:任何與您的帳戶相關聯的測試裝置
- **追蹤 Guid**:任何與您帳戶的 Guid 相關聯的追蹤

### <a name="account-details"></a>帳戶詳細資料

在 [帳戶詳細資料] 區段中，您可以看到基本資訊，例如您**帳戶類型**（「 公司 」 或 「 個別 」） 和**的驗證狀態**您的帳戶。 在您的帳戶驗證過程中，這些設定會顯示每個必要步驟，包括電子郵件驗證、 雇用驗證和業務驗證。 您也可以更新您電子郵件，並視需要重新傳送驗證。 

### <a name="publisher-ids"></a>發行者識別碼

在發行者識別碼 區段中，您可以看到您**賣方識別碼**， **MPN 識別碼**，並**發行者識別碼**。 這些值 Microsoft 指派的唯一識別您的開發人員帳戶，而且無法編輯。

### <a name="contact-info"></a>連絡資訊

在 [連絡人資訊] 區段中，您可以看到您**發行者顯示名稱**，**銷售人員的連絡資訊**（連絡人姓名、 電子郵件、 電話號碼和公司銷售者的地址），而**公司核准者**（名稱、 電子郵件和電話號碼，以核准為公司的決策的授權單位的個人）。 

### <a name="financial-details"></a>財務詳細資料

在財務詳細資料 區段中，您可以提供，或如果您發行付費應用程式、 增益集或服務，請更新您的財務資訊。 

如果您只想要列出可用的供應項目，表示您不必設定付款帳戶，或填寫任何稅務表單。 如果您稍後改變心意，並決定您想要透過 Microsoft 銷售，您可以設定您的付款帳戶，並在該時間填寫稅務表單。 

#### <a name="payout-account"></a>支出帳戶

支付帳戶是從您的銷售會被傳送至的銀行帳戶。 此銀行帳戶必須位於相同的國家/地區註冊您的合作夥伴中心帳戶的所在。

若要設定您的付款帳戶，您需要**將您的 Microsoft 帳戶建立關聯**:
1. 在**帳戶設定**下方**財務詳細資訊**區段中，選取**關聯您的 Microsoft 帳戶**。 
2. 出現提示時，登入您 Microsoft 帳戶 (MSA)。 此帳戶不能與另一個合作夥伴中心帳戶相關聯。 
3. 若要完成您的付款帳戶，完全不在合作夥伴中心記錄檔的安裝，然後登入您的 Microsoft 帳戶 （而非您的工作帳戶）。 

現在，您的 Microsoft 帳戶相關聯，若要新增的付款帳戶，您必須：
- **選擇付款方式**:銀行帳戶或 PayPal
- **新增付款資訊**:選擇帳戶類型 （支票或儲蓄）、 輸入帳戶的持有者的名稱、 帳戶號碼和路由，這可能包括數字、 帳單地址、 電話號碼或 PayPal 的電子郵件地址。 * 如需使用 PayPal，為您帳戶的付款方式，以及了解是否支援在您的上市區域的詳細資訊，請參閱 < [PayPal 資訊](https://docs.microsoft.com/windows/uwp/publish/setting-up-your-payout-account-and-tax-forms#paypal-info)。

> [!IMPORTANT]
> 變更您的付款帳戶會延遲您的付款最多一次付款週期。 因為我們需要確認帳戶變更，就像我們一樣第一次設定付款帳戶時，就會發生此延遲。 您仍然會在帳戶確認之後收到全額款項。當期付款週期未付的款項會新增至下一期。  

#### <a name="tax-profile"></a>稅金設定檔

檢閱您目前的稅金設定檔狀態，確認正確**實體類型**並**稅務憑證資訊**隨即出現。 選取 **編輯**來更新或完成任何必要的表單。

若要建立您的稅務狀態，您必須指定您的居住地的公民身分的國家/地區，並完成您的國家/地區相關聯的相應的稅務表單。

不論您國家/地區的居住地或公民資格 」，您必須填寫美國稅務表單銷售透過 Microsoft 的任何供應項目。 滿足特定的美國落地需求的合作夥伴必須填妥 IRS w-9 稅表單。 在美國以外的其他合作夥伴必須填妥 IRS W-8 表單中。 您可以填寫這些線上為您完成您的稅金設定檔。

United States 個別納稅人識別碼 （或 ITIN） 不需要從 Microsoft 接收付款，或宣告稅務條約優點。

您可以填寫並提交您的稅務表單，以電子方式在合作夥伴中心;在大部分情況下，您不需要列印，並傳送電子郵件的任何表單。

不同國家/地區和區域有不同的稅務需求。 您必須支付稅金的確切量取決於的國家/地區和區域，您用來銷售您的優惠。 Microsoft remits 銷售和使用代表您在某些國家/地區的稅金。 在過程中列出您的供應項目，將會識別這些國家/地區。 在其他國家/地區，根據您註冊其中，您可能需要為匯款銷售、 使用您的銷售，直接向本機的稅務機關稅。 此外，您會收到銷售進行可能應為 income 課稅。 我們強烈建議您連絡相關的授權單位，為您的國家/地區或區域，最適合可協助您判斷您的 Microsoft 銷售交易的適當的稅務資訊。

##### <a name="withholding-rates"></a>抑制費率
您提交您的稅務表單中的資訊會決定相應的稅務抑制速率。 預扣稅金速率僅適用於您對美國; 的銷售非美國地區位置到銷售量不會抑制。 預扣稅金的費率不盡相同，但適用於美國境外註冊大部分的開發人員，預設為 30%。 您可以減少此速率，如果您的國家/地區與美國同意所得稅條約的選擇。

##### <a name="tax-treaty-benefits"></a>稅務條約優點
如果您是在美國以外，您可以利用稅務條約優點。 這些權益而異的國家/地區國家 （地區），並且可讓您減少 Microsoft withholds 的稅金。 您可以藉由完成 W 8BEN 表單的第 ii 部分索取稅務條約優點。 我們建議您與您所在國家或區域，以判斷這些權益是否適用於您適當的資源通訊。

[進一步了解稅務詳細資料的 Windows 應用程式/遊戲開發人員和 Azure Marketplace 發行者](https://docs.microsoft.com/windows/uwp/publish/tax-details-for-paid-apps)。

#### <a name="payout-hold-status"></a>支付保留狀態

根據預設，Microsoft 會每月為基礎來傳送付款。 不過，您可以選擇暫停您的支出，這會防止您的帳戶來傳送付款。 如果您選擇暫停您的支出，我們會繼續記錄所取得，並提供詳細資料，在任何營收您**支出摘要**。 不過，我們不會傳送任何付款到您的帳戶，直到您移除保留。 

若要將您的款項放上按住不放，請前往**帳戶設定**。 底下**財務詳細資料**，請在**支出保存狀態**區段中，切換滑桿移至**上**。 您可以在任何時間，變更您的支出保留狀態，但請注意您的決定會影響下一步 的每月支出。 比方說，如果您想要保留年 4 月的支出，請確定您的支出保留狀態設為**上**年 3 月結束之前。

一旦您已設定您的支出保存狀態，以**上**，直到您切換滑桿，將會是保留的所有支出回到**關閉**。 當您這樣做時，您就會包含在下一步 的每月付款週期 （假設已符合任何適用的付款臨界值）。 例如，如果您已有您的支出上按住不放，但會要支付，產生在 6 月份，則請務必切換支出保存狀態，以**關閉**年結束前。

> [!NOTE]
> 您**支出保存狀態**選項亦適用於**所有**營收來源，並且會透過 Microsoft 合作夥伴中心，包括 Azure Marketplace、 AppSource，Microsoft Store，公告等。)。 您無法選取不同的保留狀態，針對每個營收來源。

### <a name="tracking-guids"></a>追蹤 Guid

全域唯一識別碼 (Guid) 是可用來追蹤您的 Azure 使用量的唯一參考編號 （具有 32 個十六進位數字）。 

若要建立 GUID 來進行追蹤，您應該使用 GUID 產生器。 Azure 儲存體小組已建立 [GUID 產生器表單](https://aka.ms/StoragePartners)，其會透過電子郵件傳送正確格式的 GUID 給您，且可跨不同追蹤系統重複使用。

建議您為每個產品的每個供應項目與散發通道建立唯一的 GUID。 如果您不想要分割報告，您可以讓產品的多個散發通道使用單一 GUID。

若您使用範本部署產品，且該產品在 Azure Marketplace 與 GitHub 上都有提供，您必須建立並註冊 2 個相異的 GUID：

*   Azure Marketplace 中的產品 A
*   GitHub 上的產品 A

報告是由合作夥伴值 (Microsoft 合作夥伴 ID) 與 GUID 所完成。 您也可以追蹤在更細微的層級，對齊您供應項目內的每個方案的 Guid。

如需詳細資訊，請參閱 < [Guid 常見問題集的追蹤 Azure 客戶使用情況](https://docs.microsoft.com/azure/marketplace/azure-partner-customer-usage-attribution#faq)。



## <a name="multi-user-account-management"></a>多使用者帳戶管理

合作夥伴中心會利用[Azure Active Directory](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-whatis) (Azure AD) 為多使用者帳戶存取和管理。 貴組織的 Azure AD 註冊程序的一部分是自動與您的合作夥伴中心帳戶產生關聯。 

## <a name="manage-users"></a>管理使用者

**使用者**一節的合作夥伴中心 (在**帳戶設定**) 讓您使用 Azure AD 來管理使用者、 群組和 Azure AD 應用程式，必須能夠存取您的合作夥伴中心帳戶。 請注意，若要管理使用者，您必須登入您[工作帳戶](./company-work-accounts.md)（相關聯的 Azure AD 租用戶）。 若要管理不同的工作帳戶內的使用者 / 租用戶，您必須登入並登出再登入具有的使用者身分**Manager**的權限的工作帳戶 / 租用戶。 

一旦您登入您的工作帳戶 （Azure AD 租用戶），您可以：
- [新增或移除使用者](#add-or-remove-users)
- [變更使用者密碼](#change-a-user-password)
- [新增或移除群組](#add-or-remove-users)
- [新增或移除 Azure AD 應用程式](#add-new-azure-ad-applications)
- [管理 Azure AD 應用程式的金鑰](#manage-keys-for-an-azure-ad-application)
- [定義使用者角色和權限](#define-user-roles-and-permissions)


請記住，所有的合作夥伴中心使用者 （包括群組和 Azure AD 應用程式） 必須擁有作用中的工作帳戶[Azure AD 租用戶](#manage-tenants)與您的合作夥伴中心帳戶相關聯。 

### <a name="add-or-remove-users"></a>新增或移除使用者

您的帳戶必須具備[**管理員層級**](#define-user-roles-and-permissions)權限[工作帳戶 （Azure AD 租用戶）](./company-work-accounts.md)中您要新增或編輯使用者。

#### <a name="add-existing-users"></a>新增現有的使用者

若要將使用者新增至您的合作夥伴中心帳戶，存在於您的公司[工作帳戶 （Azure AD 租用戶）](./company-work-accounts.md):

1. 移至**使用者**(底下**帳戶設定**)，然後選取**將使用者新增**。
2. 從出現的清單中選取一個或多個使用者。 您可以使用 [搜尋] 方塊搜尋特定的使用者。
* 如果您選取多個使用者新增到您的合作夥伴中心帳戶時，您必須指派它們，相同的角色或自訂的權限集。 若要新增多個使用者具有不同的角色/權限，請針對每個角色或自訂的權限集的重複這些步驟。
3.  當您完成選擇使用者，請按一下**Add selected**。
5.  在 **角色**區段中，指定所選使用者的自訂權限的角色。
6.  選取 [ **儲存**]。

#### <a name="create-new-users"></a>建立新的使用者

若要建立全新的使用者帳戶，您必須具有的帳戶[**全域管理員**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)權限。 

1. 移至**使用者**(底下**帳戶設定**)，選取**將使用者新增**，然後選擇**建立新的使用者**。
1. 針對每個新的使用者輸入名字、 姓氏和使用者名稱。 
1. 如果您想新增貴組織的目錄中有全域管理員帳戶的使用者，核取方塊標示**將此使用者的全域管理員，在您的 Azure AD 中所有目錄資源的完整控制權**。 這可讓您的公司中的所有系統管理功能的 Azure AD 使用者的完整存取。 他們將能夠新增及管理您組織的工作帳戶 （Azure AD 租用戶） 中的使用者不能在合作夥伴中心，但除非您適當的角色/權限授與帳戶。 
1. 當您核取此方塊可**將此使用者的全域管理員**，您將需要提供**密碼復原電子郵件**使用者復原他們的密碼，如有必要的。
1. 在 **群組成員資格**區段中，選取您要新增的使用者所屬任何的群組。
1. 在 **角色**區段中，指定的角色或使用者的自訂權限。
1. 選取 [ **儲存**]。

在合作夥伴中心建立新的使用者也會建立該使用者的帳戶登入的工作帳戶 （Azure AD 租用戶） 中。 變更使用者的名稱，在合作夥伴中心，將貴組織的工作帳戶 （Azure AD 租用戶） 中進行相同的變更。

#### <a name="invite-new-users-by-email"></a>透過電子郵件邀請新使用者

若要邀請的使用者，目前不是您公司的工作帳戶 （Azure AD 租用戶） 透過電子郵件的一部分，您必須具有的帳戶[**全域管理員**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)權限。 

1. 移至**使用者**(底下**帳戶設定**)，選取**將使用者新增**，然後選擇**透過電子郵件邀請使用者**。
2. 輸入一或多個電子郵件地址 （最多 10)，以逗號或分號分隔。
3. 在 **角色**區段中，指定的角色或使用者的自訂權限。
4. 選取 [ **儲存**]。

您邀請的使用者會收到電子郵件邀請，以加入您的合作夥伴中心帳戶。 在您的工作帳戶 （Azure AD 租用戶） 中，會建立新的來賓使用者帳戶。 每位使用者必須接受其邀請，才能存取您的帳戶。

如果您需要重新傳送邀請，請瀏覽**使用者**頁面上，尋找邀請的使用者清單中，選取 電子郵件地址 (或文字，指出*暫止的邀請*)。 然後，在頁面底部，選取**重新傳送邀請**。
 

> [!NOTE]
> 如果您的組織使用[目錄整合](https://go.microsoft.com/fwlink/p/?LinkID=724033)同步內部部署目錄服務與 Azure AD，您將無法在合作夥伴中心建立新的使用者、 群組或 Azure AD 應用程式。 您 （或您的內部部署目錄中的另一個系統管理員），必須建立它們直接在內部部署目錄中，您將能夠看到，並且將它們加入合作夥伴中心之前。

#### <a name="remove-a-user"></a>移除使用者

若要移除您的工作帳戶 （Azure AD 租用戶） 中的使用者，請移至**使用者**(底下**帳戶設定**)，選取您想要在最右邊的專欄中，使用此核取方塊移除使用者，然後選擇**移除**從可用的動作。 若要確認您想要移除選取的使用者，會出現快顯視窗。

#### <a name="change-a-user-password"></a>變更使用者密碼

如果有使用者需要變更其密碼，則也可以這麼本身如果您提供**密碼復原電子郵件**時建立的使用者帳戶。 您也可以遵循下列步驟，以更新使用者的密碼。 若要變更使用者的密碼，在您的公司工作帳戶 （Azure AD 租用戶） 中，您必須登入帳戶上[**全域管理員**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)權限。 請注意，這會變更您的 Azure AD 租用戶中使用者的密碼，他們用來存取合作夥伴中心及密碼。

1.  從**使用者**網頁 (底下**帳戶設定**)，選取您想要編輯的使用者帳戶的名稱。
2.  選取 **重設密碼**在頁面底部的按鈕。
3.  確認網頁隨即出現顯示使用者，包括暫時密碼的登入資訊。 請務必先列印或複製此資訊並提供給使用者，當您將無法存取的暫時密碼之後您離開此頁面。


## <a name="manage-groups"></a>管理群組

群組可讓您控制多個使用者角色和權限的全面整合。

#### <a name="add-an-existing-group"></a>新增現有的群組

若要新增的群組已存在於您組織的工作帳戶 （Azure AD 租用戶） 合作夥伴中心帳戶： 

1.  從**使用者**網頁 (底下**帳戶設定**)，選取**新增群組**。
2.  從出現的清單中選取一個或多個群組。 您可以使用 [搜尋] 方塊來搜尋特定的群組。
如果您選取多個群組新增到您的合作夥伴中心帳戶，您必須指派它們，相同的角色或自訂的權限集。 若要新增多個具有不同的角色/權限群組，請針對每個角色或自訂的權限集的重複這些步驟。
3.  當您完成選擇群組，請按一下**Add selected**。
4.  在 **角色**區段中，指定選取的群組的自訂權限的角色。 所有群組的成員都可以存取您套用至群組，不論其個別的帳戶相關聯的權限與角色的權限合作夥伴中心帳戶。
5.  選取 [ **儲存**]。

當您將現有的群組時，屬於該群組的每個使用者將能夠存取您的合作夥伴中心帳戶與群組的指派角色相關聯的權限。

#### <a name="add-a-new-group"></a>新增群組

若要新增至您的合作夥伴中心帳戶的全新的群組： 

1.  從**使用者**網頁 (底下**帳戶設定**)，選取**新增群組**。
2.  在下一步 頁面上，選取**新的群組**。
3.  輸入新群組的顯示名稱。
4.  指定的角色或群組的自訂權限。 所有群組的成員都可以存取合作夥伴中心帳戶的權限套用在這裡，不論其個別的帳戶相關聯的角色/權限。
5.  從出現的清單中選取新群組的使用者。 您可以使用 [搜尋] 方塊搜尋特定的使用者。
6.  當您完成選取使用者，請按一下**Add selected**將它們新增到新的群組。
7.  選取 [ **儲存**]。

請注意，這個新的群組將會建立您組織的工作帳戶 （Azure AD 租用戶） 中，不只是在您的合作夥伴中心帳戶。

#### <a name="remove-a-group"></a>移除群組

若要移除您的工作帳戶 （Azure AD 租用戶） 中的群組，請前往**使用者**(底下**帳戶設定**)，選取您想要在最右邊的專欄中，使用此核取方塊移除的群組，然後選擇**移除**從可用的動作。 若要確認您想要移除選取的群組，會出現快顯視窗。

## <a name="manage-azure-ad-applications"></a>管理 Azure AD 應用程式

您可以讓應用程式或服務屬於貴公司的 Azure AD 以存取您的合作夥伴中心帳戶。 

#### <a name="add-existing-azure-ad-applications"></a>新增現有的 Azure AD 應用程式 

若要新增應用程式已存在於貴公司的 Azure Active Directory: 

1.  從**使用者**網頁 (底下**帳戶設定**)，選取**加入 Azure AD 應用程式**。
2.  從出現的清單中選取一或多個 Azure AD 應用程式。 您可以使用 [搜尋] 方塊來搜尋特定 Azure AD 應用程式。 如果您選取一個以上的 Azure AD 應用程式，將新增至您的合作夥伴中心帳戶，您必須指派它們，相同的角色或自訂的權限集。 若要新增多個不同的角色/權限的 Azure AD 應用程式，請針對每個角色或自訂的權限集的重複這些步驟。
3.  當您完成選取 Azure AD 應用程式時，請按一下**Add selected**。
5.  在 **角色**區段中，指定的角色或自訂的權限，選取 Azure AD 應用程式。
6.  選取 [ **儲存**]。

#### <a name="add-new-azure-ad-applications"></a>加入新的 Azure AD 應用程式 

如果您想要授與合作夥伴中心存取權之全新的 Azure AD 應用程式帳戶，您可以建立一種**使用者**一節。 請注意，這會建立新的帳戶，在您的公司工作帳戶 （Azure AD 租用戶），不只是在您的合作夥伴中心帳戶。 如果您主要用於此 Azure AD 應用程式合作夥伴中心驗證，並不需要直接存取的使用者，您可以輸入任何有效的位址，如**回覆 URL**並**應用程式識別碼 URI**，只要做為這些值不會使用您的目錄中任何其他 Azure AD 應用程式。

1.  從**使用者**網頁 (底下**帳戶設定**)，選取**加入 Azure AD 應用程式**。
2.  在下一步 頁面上，選取**新增 Azure AD 應用程式**。
3.  請輸入**回覆 URL**新的 Azure AD 應用程式。 這是使用者可以登入並使用 Azure AD 應用程式 （有時也稱為應用程式 URL 或登入 URL） 的 URL。 **回覆 URL**不能超過 256 個字元，而且必須是唯一在您的目錄。
4.  請輸入**應用程式識別碼 URI**新的 Azure AD 應用程式。 這是單一登入要求傳送至 Azure AD 時，會顯示 Azure AD 應用程式的邏輯識別碼。 請注意，**應用程式識別碼 URI**必須是唯一的目錄中的每個 Azure AD 應用程式。 此識別碼不得超過 256 個字元。 如需應用程式識別碼 URI 的詳細資訊，請參閱[整合應用程式與 Azure Active Directory](https://docs.microsoft.com/azure/active-directory/develop/quickstart-modify-supported-accounts#change-the-application-registration-to-support-different-accounts)。
5.  在 **角色**區段中，指定的角色或 Azure AD 應用程式的自訂權限。
6.  選取 [ **儲存**]。

您加入或建立 Azure AD 應用程式之後，您可以返回**使用者**區段，然後選取 檢閱設定，讓應用程式，包括租用戶識別碼、 用戶端識別碼、 回覆 URL 和應用程式識別碼 URI 的應用程式名稱。

#### <a name="remove-an-application"></a>移除應用程式

若要移除您的工作帳戶 （Azure AD 租用戶） 中的應用程式，請前往**使用者**(底下**帳戶設定**)，選取您想要在最右邊的專欄中，使用此核取方塊移除的應用程式然後選擇**移除**從可用的動作。 若要確認您想要移除選取的應用程式，會出現快顯視窗。

#### <a name="manage-keys-for-an-azure-ad-application"></a>管理 Azure AD 應用程式的金鑰

如果您的 Azure AD 應用程式讀取和寫入 Microsoft Azure AD 中的資料，它將需要金鑰。 您可以藉由編輯其資訊在合作夥伴中心建立為 Azure AD 應用程式的金鑰。 您也可以移除不再需要的索引鍵。

1.  從**使用者**網頁 (底下**帳戶設定**)，選取 Azure AD 應用程式的名稱。 您會看到所有作用中的索引鍵，Azure AD 應用程式，包括在其中建立金鑰和它何時到期的日期。 
2. 若要移除不再需要的索引鍵，選取**移除**。
3.  若要加入新的金鑰，請選取**新增新的金鑰**。
4.  您會看到畫面顯示**用戶端識別碼**並**鍵值**。 請務必先列印或複製這項資訊，因為您將無法再存取它之後您離開此頁面。
4.  如果您想要建立多個索引鍵，選取**新增另一個索引鍵**。

## <a name="manage-tenants"></a>管理租用戶

也請參考本文件中，您 「 工作帳戶 」 的 Azure Active Directory (AD) 租用戶是貴組織的表示法在 Azure 入口網站中設定，並幫助您管理您的內部 Microsoft 雲端服務的特定執行個體和外部的使用者。 如果您的組織已訂閱 Microsoft 雲端服務，例如 Azure、 Microsoft Intune 或 Office 365 所建立的 Azure AD 租用戶。 

使用合作夥伴中心，您可以設定多個租用戶。 任何擁有使用者**Manager**合作夥伴中心帳戶中的角色必須新增和移除帳戶的 Azure AD 租用戶的選項。  

### <a name="add-an-existing-tenant"></a>新增現有的租用戶

若要關聯您的合作夥伴中心帳戶中的另一個 Azure AD 租用戶：

1.  從**租用戶**網頁 (底下**帳戶設定**)，選取**將另一個 Azure AD 租用戶關聯**。
2. 輸入您想要關聯的租用戶的 Azure AD 認證。
3. 檢閱您的 Azure AD 租用戶的組織和網域名稱。 若要完成此關聯，請選取**確認**。

如果關聯的不成功，您將則是可以新增和管理帳戶的使用者**使用者**在合作夥伴中心內的一節。

### <a name="create-a-new-tenant"></a>建立新的租用戶

若要建立品牌新的 Azure AD 租用戶與您的合作夥伴中心帳戶：

1.  從**租用戶**網頁 (底下**帳戶設定**)，選取**建立一個新的 Azure AD 租用戶**。
2. 輸入新的 Azure AD 的目錄資訊：
    - **網域名稱**：我們將使用您的 Azure AD 網域，連同的唯一名稱 」。 onmicrosoft.com"。 比方說，如果您輸入 「 範例 」，您的 Azure AD 網域會是"example.onmicrosoft.com 」。
    - **連絡人電子郵件**:電子郵件地址，其中我們可以連絡您關於您的帳戶如有必要。
    - **全域管理員使用者帳戶資訊**:第一個名稱，最後一個名稱、 使用者名稱和您想要使用新的全域系統管理員帳戶的密碼。
3. 選取 **建立**確認新的網域和帳戶資訊。
4. 登入新的 Azure AD 全域系統管理員使用者名稱和密碼，才能開始[新增和管理使用者](#manage-users)。

如需有關如何建立新的租用戶，在 Azure 入口網站中，而不是透過合作夥伴中心入口網站的詳細資訊，請參閱[Azure Active Directory 中建立新的租用戶](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-access-create-new-tenant)。

### <a name="remove-a-tenant"></a>移除租用戶

若要移除您的合作夥伴中心帳戶中的租用戶，其名稱的上找到**租用戶**網頁 (在**帳戶設定**)，然後選取**移除**。 系統會提示您確認您想要移除租用戶。 一旦您這樣做時，該租用戶中的使用者無法登入合作夥伴中心帳戶，並將移除任何您已設定為這些使用者的權限。

當您移除租用戶時，已新增到合作夥伴中心帳戶，從該租用戶的所有使用者將不再能夠登入帳戶。

> [!TIP]
> 如果您目前登入合作夥伴中心使用相同的租用戶中的帳戶，您無法移除租用戶。 若要移除租用戶，您必須登入合作夥伴中心**Manager**與帳戶相關聯的其他租用戶。 如果只有一個租用戶帳戶相關聯，該租用戶只有開啟該帳戶的 Microsoft 帳戶登入之後移除。
